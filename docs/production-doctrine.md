# Doutrina de Produção Hub Jogos: Visual-First & Fun-First

Esta doutrina define a regra de ouro para o desenvolvimento de jogos no Hub: **O visual e a diversão não são opcionais, são o portão de entrada para a credibilidade política.**

## 1. A Regra Visual-First
Um jogo classificado como **Flagship (Carro-chefe)** ou de prioridade pública deve obrigatoriamente incluir:
- **Mundo Visível**: Um tabuleiro, mapa ou cenário que represente o território, não apenas menus.
- **Verbos Ativos**: O jogador deve "fazer" algo no mundo (despachar, construir, mover), não apenas selecionar opções de texto.
- **Estado Reativo**: O mundo deve mudar visivelmente (luzes, cores, animações) em resposta às decisões.
- **Planejamento de Assets**: Assets visuais devem ser planejados desde a pré-produção.
- **Diversão nos Primeiros 20 Segundos**: O jogador deve entender o "prazer" do jogo imediatamente.

## 2. Reclassificação Estratégica
Os jogos do Hub são divididos em três classes de produção:

| Classe | Definição | Formatos Permitidos | Exigência Visual |
| :--- | :--- | :--- | :--- |
| **Classe A** | Flagships / Carros-chefe | Arcade, Simulação Urbana, Estratégia, Tycoon | Alta (Assets, Spectacle, Juice) |
| **Classe B** | Suporte / Quickplay | Quizzes Visuais, Branching Stories com cenários | Média (Ilustrações, Icons, UI rica) |
| **Classe C** | Lab / Protótipos | Experimentos de sistema, Calculadoras políticas | Funcional (Foco na lógica) |

> [!IMPORTANT]
> Projetos focados apenas em cards ou decisões de texto nunca serão **Classe A**. Eles devem servir como suporte ou introdução para experiências de **Classe A**.

## 3. Portão "Toy Factor"
Antes de avançar para a fase pública, todo jogo deve passar pelo teste do Fator Brinquedo:
1. **É prazeroso tocar?** (Hover, click, scroll têm feedback satisfatório?)
2. **O mundo reage visivelmente?** (Há animação ou mudança de estado no cenário?)
3. **Um jogador comum chamaria isso de jogo?** (Ou parece um formulário inteligente?)
4. **O screenshot é convidativo?** (Comunica tensão ou progresso visualmente?)
5. **Há diversão antes da explicação?** (A mecânica básica é intuitiva?)

## 4. Alvos de Gênero Visual
- **Arcade**: Espetáculo, urgência, ameaças legíveis.
- **Simulação**: Reação visível da cidade/mundo.
- **Estratégia**: Tabuleiro tático com vida e consequências.
- **RPG/Narrativa**: Presença de cena, não apenas cards de diálogo.

## 5. Template de Planejamento de Assets
Todo novo projeto Classe A deve preencher:
- [ ] **Background/Mundo**: Qual é o cenário base?
- [ ] **Hotspots/Zonas**: Quais áreas são interativas?
- [ ] **Unidades/Atores**: O que se move no mundo?
- [ ] **Props de Infraestrutura**: Detalhes que dão vida (fios, postes, lixo, árvores).
- [ ] **Arte de HUD/UI**: Ícones e painéis temáticos.
- [ ] **FX de Feedback**: Partículas, flashes, tremores (Juice).
- [ ] **Visual de Resultado**: Como a vitória/derrota é mostrada no mundo?
